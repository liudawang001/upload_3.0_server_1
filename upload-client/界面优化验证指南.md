# JavaFX界面优化验证指南

## 📋 **完整操作步骤**

### 🔧 **1. 编译和打包步骤**

#### **步骤1.1：关闭正在运行的应用程序**
```bash
# 关闭所有Java进程（如果有应用程序正在运行）
taskkill /F /IM java.exe
taskkill /F /IM javaw.exe

# 或者手动关闭JavaFX应用程序窗口
```

#### **步骤1.2：清理和重新编译**
```bash
# 进入项目目录
cd upload-client

# 清理之前的构建文件
mvn clean

# 重新编译项目以应用最新的界面修改
mvn compile -q

# 验证编译成功（应该没有错误输出）
```

#### **步骤1.3：打包应用程序（可选）**
```bash
# 如果需要生成可执行文件
mvn package -Dmaven.test.skip=true
```

---

### 🚀 **2. 启动应用程序**

#### **方法1：使用Maven直接运行（推荐）**
```bash
# 使用Maven运行主类
mvn exec:java -Dexec.mainClass="com.mls.upload.client.MainApplication"

# 或者使用备用主类
mvn exec:java -Dexec.mainClass="com.mls.upload.client.UploadClientApplication"
```

#### **方法2：使用Java命令运行**
```bash
# 确保在upload-client目录下
java -cp "target/classes;target/lib/*" com.mls.upload.client.MainApplication
```

#### **方法3：使用启动脚本（如果存在）**
```bash
# 查看是否有启动脚本
dir *.bat

# 如果有start-client.bat或类似脚本
start-client.bat
```

#### **启动注意事项**
- ✅ 确保Java 8或更高版本已安装
- ✅ 确保JavaFX运行时可用
- ✅ 确保没有其他实例正在运行
- ✅ 检查控制台输出是否有错误信息

### 🔧 **4. 故障排除**

#### **4.1 启动问题**

**问题：应用程序无法启动**
```bash
# 解决方案1：检查Java版本
java -version

# 解决方案2：检查JavaFX可用性
java -cp "target/classes" com.mls.upload.client.MainApplication

# 解决方案3：查看详细错误信息
mvn exec:java -Dexec.mainClass="com.mls.upload.client.MainApplication" -X
```

**问题：找不到主类**
```bash
# 解决方案：重新编译
mvn clean compile

# 检查主类是否存在
dir target\classes\com\mls\upload\client\*.class
```

#### **4.2 界面显示异常**

**问题：界面元素重叠或显示不正常**
```
检查项目：
1. 确认FXML文件已正确更新
2. 检查CSS样式文件是否存在
3. 验证字体是否可用
4. 确认屏幕分辨率和DPI设置
```

**问题：底部状态栏仍然被截断**
```
解决方案：
1. 检查窗口实际尺寸
2. 确认最小高度设置生效
3. 验证FXML prefHeight设置
4. 检查是否有其他CSS覆盖设置
```

#### **4.3 常见错误及解决方法**

**错误：JavaFX runtime components are missing**
```bash
# 解决方案：安装JavaFX或使用包含JavaFX的JDK
# 或者添加JavaFX模块路径
java --module-path "path/to/javafx/lib" --add-modules javafx.controls,javafx.fxml
```

**错误：FXML文件加载失败**
```bash
# 解决方案：检查资源文件
dir target\classes\fxml\main.fxml

# 重新处理资源
mvn process-resources
```

**错误：CSS样式不生效**
```bash
# 解决方案：检查CSS文件
dir target\classes\css\main.css

# 清理并重新编译
mvn clean compile
```
