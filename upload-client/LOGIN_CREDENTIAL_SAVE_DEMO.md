# 用户登录信息自动保存功能演示

## 功能概述

本功能实现了前端用户登录信息的自动保存，用户可以选择是否记住密码，下次启动应用程序时自动填充上次成功登录的用户凭据。

## 核心特性

### ✅ **安全性**
- **密码加密存储**：使用AES加密算法，密码不以明文形式保存
- **动态密钥生成**：每次首次使用时生成唯一的加密密钥
- **用户选择权**：提供"记住密码"选项，用户可自主选择是否保存

### ✅ **用户体验**
- **自动填充**：应用启动时自动填充上次成功登录的凭据
- **智能记忆**：只保存最后一次成功登录的用户信息
- **一键清除**：提供清除保存信息的功能

### ✅ **数据管理**
- **本地存储**：使用Java Preferences API进行本地存储
- **多用户支持**：支持不同用户的凭据覆盖保存
- **数据验证**：严格验证空值和无效数据

## 技术实现

### 核心类：CredentialManager

**主要功能**：
- 安全地保存和读取用户登录凭据
- 使用AES加密算法保护密码安全
- 管理用户的"记住密码"偏好设置

**关键方法**：

```java
// 保存用户凭据
public void saveCredentials(String username, String password, boolean rememberPassword)

// 读取保存的凭据
public SavedCredentials loadCredentials()

// 清除保存的凭据
public void clearCredentials()

// 检查是否有保存的凭据
public boolean hasCredentials()
```

### 加密安全

**加密算法**：AES (Advanced Encryption Standard)
**密钥管理**：
- 首次使用时自动生成128位密钥
- 密钥使用Base64编码存储在Preferences中
- 每个应用实例使用独立的加密密钥

**安全特性**：
- 密码明文不会出现在任何日志中
- 加密密钥与加密数据分离存储
- 支持密钥重新生成和数据迁移

### 数据存储

**存储位置**：Java Preferences API
- Windows: 注册表 `HKEY_CURRENT_USER\Software\JavaSoft\Prefs\com\mls\upload\client\credentials`
- Linux: `~/.java/.userPrefs/com/mls/upload/client/credentials`
- macOS: `~/Library/Preferences/com.apple.java.util.prefs.plist`

**存储内容**：
```
rememberPassword: boolean    // 是否记住密码
lastUsername: String        // 最后登录的用户名
encryptedPassword: String   // 加密后的密码(Base64)
encryptionKey: String       // 加密密钥(Base64)
```

## 使用流程

### 首次登录

1. **用户输入凭据**：输入用户名和密码
2. **选择记住密码**：勾选"记住密码"复选框（可选）
3. **点击登录**：系统验证凭据
4. **登录成功**：如果勾选了"记住密码"，系统自动保存凭据
5. **下次启动**：应用自动填充保存的用户名和密码

### 后续使用

1. **应用启动**：自动检查是否有保存的凭据
2. **自动填充**：如果有保存的凭据，自动填充用户名和密码字段
3. **复选框状态**：恢复用户的"记住密码"选择状态
4. **一键登录**：用户可直接点击登录，无需重新输入

### 更换用户

1. **清空字段**：手动清空用户名和密码字段
2. **输入新凭据**：输入新用户的登录信息
3. **选择保存**：勾选"记住密码"（如果需要）
4. **登录成功**：新用户凭据将覆盖之前保存的信息

## 安全考虑

### 密码保护

- **不明文存储**：密码经过AES加密后存储
- **密钥隔离**：加密密钥与密码数据分开存储
- **动态生成**：每个应用实例使用独立的加密密钥

### 数据验证

- **空值检查**：严格验证用户名和密码不为空

- **异常处理**：完善的错误处理和恢复机制

### 用户控制

- **随时清除**：用户可随时清除保存的凭据
- **透明操作**：所有操作都有明确的用户提示

## 故障排除

### 常见问题

1. **凭据无法保存**：
   - 检查应用程序权限
   - 确认Preferences API可用性
   - 查看错误日志

2. **自动填充失效**：
   - 检查保存的凭据是否完整
   - 验证加密密钥是否正确
   - 重新保存凭据

3. **密码解密失败**：
   - 清除所有保存的数据
   - 重新生成加密密钥
   - 重新保存凭据

### 日志信息

系统会记录以下关键操作：
- 凭据保存成功/失败
- 凭据读取成功/失败
- 加密密钥生成
- 数据清除操作
