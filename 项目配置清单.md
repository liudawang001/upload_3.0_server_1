## 4. 部署注意事项

### 4.1 高优先级配置项（必须修改）

1. **文件存储路径**：
   - 花型图路径：`app.file.upload.flower-path`
   - 配色图路径：`app.file.upload.color-path`
   - 需要根据服务器实际存储结构调整

2. **数据库连接配置**：
   - 所有环境的数据库URL、用户名、密码
   - 确保服务器能够访问数据库

3. **网络地址配置**：
   - 客户端服务器连接地址
   - Docker特征提取服务地址
   - 缓存刷新目标地址

4. **Java代码硬编码**：
   - ConfigUtil类中的默认URL
   - FileService类中的默认路径

### 4.2 中优先级配置项（建议修改）

1. **日志文件路径**：
   - 根据服务器日志管理策略调整
   - 生产环境建议使用绝对路径

2. **临时文件路径**：
   - 确保有足够的磁盘空间
   - 考虑定期清理策略

3. **连接池配置**：
   - 根据服务器性能和并发需求调整

### 4.3 低优先级配置项（可选修改）

1. **显示信息**：
   - 启动脚本中的提示信息
   - 可保持默认或根据需要调整

2. **性能参数**：
   - 超时时间、重试次数等
   - 可根据网络环境优化

### 4.4 部署建议

1. **配置文件管理**：
   - 使用环境变量或外部配置文件
   - 避免在代码中硬编码敏感信息

2. **路径规范**：
   - 使用绝对路径避免相对路径问题
   - 确保目录权限正确

3. **网络配置**：
   - 确认防火墙和端口开放
   - 测试各服务间的网络连通性

4. **备份策略**：
   - 配置文件版本控制
   - 重要数据定期备份

5. **监控和日志**：
   - 配置适当的日志级别
   - 设置日志轮转和清理策略

## 8. 常见问题和解决方案

### 8.1 路径相关问题
**问题**：文件上传失败，提示路径不存在
**解决**：
1. 检查配置文件中的路径是否正确
2. 确保目录存在且有写入权限
3. 在Linux系统中注意路径分隔符

### 8.2 数据库连接问题
**问题**：数据库连接失败
**解决**：
1. 检查数据库服务是否启动
2. 验证IP地址、端口、用户名、密码
3. 确认防火墙设置
4. 检查数据库用户权限

### 8.3 网络连接问题
**问题**：客户端无法连接服务端
**解决**：
1. 检查服务端是否正常启动
2. 验证IP地址和端口配置
3. 确认防火墙和网络策略
4. 测试网络连通性

### 8.4 Docker服务问题
**问题**：特征提取功能异常
**解决**：
1. 检查Docker服务是否运行
2. 验证Docker服务地址配置
3. 查看Docker服务日志
4. 可临时禁用特征提取功能
