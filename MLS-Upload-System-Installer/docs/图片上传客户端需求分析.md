### 1.1 项目概述

开发一个基于Windows系统的图片信息上传客户端，用于将本地图片上传至远程服务器，并记录相关元数据信息到MySQL数据库中。系统需提供用户登录、文件夹选择、图片分类、上传进度显示及日志记录等功能。

本项目分为客户端和服务端，客户端采用JavaFX，要求可以打包成exe文件，直接在windows7以上的系统上运行。

在上传花型图的同时，需要调用图片特征提取的docker，将图片特征存放在数据库中。

上传配色图时，不需要提取特征。

### 1.2 功能需求

#### 1.2.1 用户登录

- 输入服务器地址、用户名和密码进行身份验证，点击登录按钮验证。

- 记录登录日志（成功/失败）。

- 数据库表对应pic_user

- 表结构（表已经建好）：

- CREATE TABLE `pic_user` (
    `id` int NOT NULL AUTO_INCREMENT COMMENT '用户ID',
    `username` varchar(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '用户名',
    `password` varchar(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '密码',
    `nickname` varchar(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL COMMENT '昵称',
    `create_time` datetime DEFAULT NULL COMMENT '创建日期',
    `updata_time` datetime DEFAULT NULL,
    PRIMARY KEY (`id`) USING BTREE
  ) ENGINE=InnoDB AUTO_INCREMENT=8 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci ROW_FORMAT=DYNAMIC COMMENT='用户表';
  
- 原系统密码存储与解析格式：
  **控制器接收**（AuthController.java）：

  @PostMapping("/register")
  public ApiResponse<RegisterResponse> register(@RequestBody RegisterRequest registerRequest)
  **请求DTO结构**（RegisterRequest.java）：
  @Data
  public class RegisterRequest {
      private String username;
      private String password;        // 明文密码
      private String confirmPassword; // 确认密码
      private String nickname;
  }
  **密码编码器配置**（PasswordConfig.java）：
  @Configuration
  public class PasswordConfig {
      @Bean
      public PasswordEncoder passwordEncoder() {
          return new BCryptPasswordEncoder();  // 使用BCrypt算法
      }
  }
  **密码加密过程**（AuthService.java）：private PicUser createNewUser(RegisterRequest registerRequest) {

    PicUser user = new PicUser();

    user.setUsername(registerRequest.getUsername());

    

    // 🔑 关键步骤：使用BCrypt加密密码

    user.setPassword(passwordEncoder.encode(registerRequest.getPassword()));

    

    // 设置其他字段...

    return user;

  }
  **SQL插入语句**（PicUserMapper.xml）：<insert id="insertUser" parameterType="com.mls.pic.entity.PicUser" useGeneratedKeys="true" keyProperty="id">
      INSERT INTO pic_user (username, password, nickname, create_time, updata_time)
      VALUES (#{username}, #{password}, #{nickname}, #{createTime}, #{updataTime})
  </insert>

#### 1.2.2 图片分类选择

- 用户可选择图片类别：花型图或配色图。
- 如果用户选择花型图，则直接上传图片到服务器花型图文件夹：G:\A_Projects\Pic_Manage\mls_image
- 如果用户选择配色图，将图片上传到配色图文件夹：G:\A_Projects\Pic_Manage\mls_image\color_scheme
- 在图片上传的同时，将文件名与上传日期存放到表image_upload表中（表已创建）。
- image_upload表结构如下：
  id --- id号（自动递增）
  filename --- VARCHAR（100）--- 文件名
  image_type --- VARCHAR（100）--- 图片类型（1表示花型图，2表示配色图）
  create_time --- datetime --- 创建时间
  username --- VARCHAR（100）--- 用户名（记录哪一个用户上传的文件）

#### 1.2.3 上传表格

- 用户点击上传表格组件区域，完成表格上传。
- 上传表格后，要在上传组件显示表格名称。
- 上传表格后，系统自动解析表格，将表格中的商品编号一栏，显示在右侧日志中。
- 表格中的第一行标题与表pic_info相对应。
- 上传表格后，日志区的商品编号栏，显示表格中含有的所有商品编号。

#### 1.2.4 文件夹选择

- 用户可选择本地文件夹，系统显示该文件夹下的图片数量。
- 支持多次选择，每次选择后更新图片数量。
- 选择文件夹后，日志区域图片名一栏显示图片名，表格中的商品编号要与图片名相对应。（图片名的命名方式一般是商品编号+.jpg）
- 如果商品编号和图片名相对应，则日志区的比对状态显示绿色字体“比对成功”，如果商品编号存在，图片名不存在，显示红色字体”缺少图片“；如果商品编号不存在，图片名存在，显示红色字体”缺少数据“。

#### 1.2.5 信息上传

- 支持多张图片批量上传。

- 显示总体上传进度（已上传数量 / 文件夹中图片总数量）。

- 实时显示每张图片的上传状态（成功/失败）。

- 记录上传日志，更新日志区域的上传状态。

- 注意：只上传比对状态为”比对成功“的数据。

- 在花型图图片上传时，需要检查文件夹color_scheme中是否存在同名的图片，如果有，就覆盖替换掉原来的图片。

### 1.3 非功能需求

- 系统需运行在Windows平台。
- 支持断点续传（可选，但建议）。
- 界面友好，操作简便。
- 具备一定的错误处理能力（如网络异常、数据库连接失败等）。

### 1.4 数据库需求

使用MySQL数据库存储以下信息：

- 用户信息（用户名、密码等）。
- 上传记录（图片名称、大小、类别、上传时间、状态等）。
- 系统日志（登录日志、操作日志等）。
