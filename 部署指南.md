# 木林森图片上传系统部署指南

## 📋 系统概述

木林森图片上传系统是一个基于JavaFX客户端和Spring Boot服务端的企业级图片批量上传解决方案。

### 🏗️ 系统架构
- **客户端**：JavaFX 8 桌面应用程序（Windows）
- **服务端**：Spring Boot 2.1.18 REST API服务
- **数据库**：MySQL 8.0
- **通信协议**：HTTP + JSON + Base64图片传输

## 🔧 环境要求

### 服务端环境
- **操作系统**：Windows/Linux/macOS
- **Java版本**：JDK 8 或更高版本
- **数据库**：MySQL 8.0
- **内存**：最小2GB，推荐4GB
- **磁盘空间**：最小1GB可用空间

### 客户端环境
- **操作系统**：Windows 7/8/10/11
- **Java版本**：JDK 8 或更高版本（如果使用jar方式运行）
- **内存**：最小1GB，推荐2GB
- **磁盘空间**：最小500MB可用空间

## 🚀 部署步骤

### 第一步：数据库准备

1. **安装MySQL 8.0**
   ```bash
   # 创建数据库
   CREATE DATABASE pic_manage CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   
   # 创建用户（可选）
   CREATE USER 'pic_user'@'%' IDENTIFIED BY 'your_password';
   GRANT ALL PRIVILEGES ON pic_manage.* TO 'pic_user'@'%';
   FLUSH PRIVILEGES;
   ```

2. **配置数据库连接**
   - 编辑 `upload-server/src/main/resources/application.yml`
   - 修改数据库连接信息：
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://47.110.46.27:3306/pic_manage
       username: your_username
       password: your_password
   ```

### 第二步：服务端部署

1. **检查Java环境**
   ```bash
   java -version
   # 确保显示Java 8或更高版本
   ```

2. **启动服务端**
   
   **方式一：使用启动脚本（推荐）**
   ```bash
   # Windows
   cd upload-server
   start-server.bat
   
   # Linux/macOS
   cd upload-server
   java -jar target/upload-server.jar
   ```
   
   **方式二：直接运行jar文件**
   ```bash
   cd upload-server
   java -jar target/upload-server.jar
   ```

3. **验证服务端启动**
   - 打开浏览器访问：`http://127.0.0.1:8081/api/auth/health`
   - 应该看到JSON响应：`{"code":200,"message":"认证服务运行正常",...}`

### 第三步：客户端部署

编译打包客户端：
```bash
cd upload-client
mvn clean package -DskipTests   或者
mvn clean package "-Dmaven.test.skip=true"
```

1. **方式一：使用exe文件（推荐）**
   ```bash
   # 双击运行或使用启动脚本
   cd upload-client
   start-client.bat
   ```

2. **方式二：使用jar文件**
   ```bash
   cd upload-client
   java -jar target/upload-client.jar
   ```

## ⚙️ 配置说明

### 服务端配置

**application.yml 主要配置项：**
```yaml
server:
  port: 8081                    # 服务端口
  servlet:
    context-path: /api          # API上下文路径

spring:
  datasource:
    url: jdbc:mysql://47.110.46.27:3306/pic_manage
    username: your_username     # 数据库用户名
    password: your_password     # 数据库密码
    
  servlet:
    multipart:
      max-file-size: 50MB      # 单文件最大大小
      max-request-size: 500MB  # 请求最大大小

logging:
  level:
    com.mls.upload.server: DEBUG  # 日志级别
```

### 客户端配置

**application.properties 主要配置项：**
```properties
# 服务端配置
server.base.url=http://127.0.0.1:8081
server.timeout=30000

# 上传配置
upload.batch.size=10
upload.thread.pool.size=5
upload.retry.count=3

# 界面配置
ui.window.width=1000
ui.window.height=700
ui.window.resizable=true
```

## 🔍 故障排除

### 常见问题

1. **服务端启动失败**
   - 检查Java版本：`java -version`
   - 检查端口占用：`netstat -an | findstr 8081`
   - 检查数据库连接配置

2. **客户端连接失败**
   - 确认服务端已启动
   - 检查防火墙设置
   - 验证服务端地址配置

3. **数据库连接失败**
   - 检查MySQL服务状态
   - 验证数据库用户权限
   - 确认网络连通性

### 日志文件位置

**服务端日志：**
- 控制台输出
- 日志文件：`logs/upload-server.log`

**客户端日志：**
- 控制台输出
- 日志文件：`logs/upload-client.log`
- 错误日志：`logs/upload-client-error.log`

## 🔒 安全建议

1. **数据库安全**
   - 使用强密码
   - 限制数据库访问IP
   - 定期备份数据

2. **网络安全**
   - 配置防火墙规则
   - 使用HTTPS（生产环境）
   - 限制API访问

3. **应用安全**
   - 定期更新依赖库
   - 监控系统日志
   - 设置访问控制
